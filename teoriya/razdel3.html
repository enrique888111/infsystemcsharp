Раздел 3

Файлы. Виды файлов

В C# работа с файлами включает в себя множество различных операций, таких как чтение, запись, создание, удаление и т. д. Основными видами файлов, с которыми вы столкнетесь при работе на C#, являются:

  • Текстовые файлы: Это файлы, содержащие текстовую информацию. В C# для работы с текстовыми файлами используются классы System.IO.File и System.IO.StreamReader для чтения из файла и System.IO.StreamWriter для записи в файл.
  • Бинарные файлы: Это файлы, содержащие данные в бинарном формате. Они могут быть использованы для хранения структурированных данных, таких как изображения, звуковые файлы и другие не-текстовые данные. Для работы с бинарными файлами также используются классы из System.IO, но обычно с использованием более низкоуровневых методов, таких как BinaryReader и BinaryWriter.
  • JSON файлы: JSON (JavaScript Object Notation) - это формат обмена данными, который широко используется для сериализации и десериализации структурированных данных. В C# вы можете использовать библиотеку Newtonsoft.Json (также известную как Json.NET) для работы с JSON файлами.
  • XML файлы: XML (eXtensible Markup Language) - это другой формат для представления структурированных данных. Он часто используется для обмена данными между различными приложениями. В C# для работы с XML файлами также можно использовать встроенные классы System.Xml, такие как XmlDocument и XmlTextWriter, а также XmlSerializer.
  • Конфигурационные файлы: Это файлы, содержащие настройки или параметры для приложения. В .NET Framework конфигурационные файлы часто имеют формат XML и имеют расширение .config, например app.config или web.config.
  • Ресурсы и встроенные ресурсы: Ресурсы могут быть встроены в исполняемый файл или храниться отдельно в файлах, обычно с расширением .resx. В C# вы можете использовать ресурсы для локализации приложения или хранения строковых констант и других ресурсов.
  • Изображения и мультимедиа файлы: Это файлы, содержащие изображения, видео, аудио и другие мультимедийные данные. Для работы с изображениями в C# вы можете использовать классы из System.Drawing или System.Windows.Media.Imaging.

Файлы и сериализация объектов

В C#, сериализация объектов используется для преобразования объектов в поток байтов, который может быть сохранен в файле или передан по сети, а затем восстановлен обратно в объект. Это полезно, например, при сохранении состояния приложения, обмене данными между приложениями или хранении объектов в базе данных. Существует несколько способов сериализации объектов в C#. Одним из наиболее популярных способов является использование встроенного механизма сериализации, предоставляемого .NET Framework.

Библиотеки динамической компоновки

Динамическая компоновка в C# позволяет создавать, компилировать и выполнять код во время выполнения программы. Это полезно, например, для создания расширяемых систем, плагинов, скриптов или для работы с динамически создаваемыми пользовательскими интерфейсами.

Вот несколько библиотек и инструментов, предоставляющих функциональность динамической компоновки в C#:

  • Roslyn: Это набор инструментов и компилятор C#, который позволяет анализировать и компилировать код на лету.
  • System.Reflection: Это пространство имен в .NET Framework, которое позволяет получать информацию о типах и их членах, создавать экземпляры типов и вызывать их методы динамически.
  • DynamicMethod: Этот класс позволяет создавать и компилировать методы на лету во время выполнения программы.
  • Mono.CSharp: Это библиотека, предоставляющая компилятор C# на лету для выполнения кода в среде .NET.
  • ScriptCS: Это инструмент, позволяющий писать и выполнять скрипты на C# без явной компиляции, используя Roslyn.

Обобщения. Обобщенные классы. Иерархии обобщенных классов

Обобщения (Generics) в C# предоставляют возможность создавать обобщенные типы и методы, которые могут работать с различными типами данных, обеспечивая безопасность типов во время компиляции. Обобщения позволяют писать более гибкий и эффективный код.

Обобщенные классы (Generic classes) - это классы, которые могут быть параметризованы типами данных. При определении обобщенного класса вы указываете типы параметров в угловых скобках. Иерархия обобщенных классов в C# - это последовательность классов, которые связаны отношением наследования и используют обобщения. Иерархия обобщенных классов может быть более сложной, с множеством уровней наследования и множеством параметров типа. Она обеспечивает гибкость и повторное использование кода для работы с различными типами данных.

Коллекции. Типы коллекций. Необобщенные коллекции. Обобщенные коллекции

В программировании коллекции представляют собой структуры данных, предназначенные для хранения и управления набором объектов. В C# существует несколько типов коллекций, которые можно разделить на необобщенные и обобщенные. Необобщенные коллекции:

  • ArrayList: Это коллекция, которая может содержать элементы любого типа данных. Однако, поскольку она не является обобщенной, при доступе к элементам требуется явное приведение типов, что может привести к ошибкам времени выполнения.
  • Hashtable: Это структура данных, представляющая собой таблицу соответствия ключей и значений. Ключи и значения могут быть любого типа данных.
  • Queue: Очередь, которая работает по принципу "первым пришел - первым вышел" (FIFO).
  • Stack: Стек, который работает по принципу "последним пришел - первым вышел" (LIFO).
Обобщенные коллекции:
  • List: Динамический массив, который может содержать элементы одного типа T. Обеспечивает быстрый доступ к элементам по индексу.
  • Dictionary: Представляет коллекцию пар ключ-значение, где каждый ключ уникален.
  • HashSet: Коллекция, которая хранит уникальные элементы, позволяя быстро проверять наличие элемента в коллекции.
  • Queue и Stack: Обобщенные версии очереди и стека, соответственно.
Обобщенные коллекции предпочтительны, поскольку они обеспечивают безопасность типов во время компиляции и избавляют от необходимости явного приведения типов. Они также предоставляют более эффективную реализацию и могут быть более удобными в использовании.

Принципы работы с XML-данными. Класс XmlDocument. Класс XPathNavigator

XML (eXtensible Markup Language) - это универсальный формат для обмена данными между различными системами. В C# существует несколько способов работы с XML-данными, включая классы XmlDocument и XPathNavigator. XmlDocument: Класс XmlDocument представляет документ XML в виде иерархического дерева узлов. Он позволяет загружать, создавать, изменять и сохранять XML-документы. XPathNavigator: Класс XPathNavigator предоставляет навигацию и доступ к данным в XML-документе с использованием языка запросов XPath. Он позволяет выполнять различные операции с XML-данными, такие как поиск, фильтрация, выборка и т. д., используя мощные возможности XPath.

Основы многопоточного программирования

Многопоточное программирование в C# позволяет создавать приложения, которые выполняют несколько задач одновременно. Это особенно полезно для улучшения производительности и реактивности приложений. Вот основные концепции многопоточного программирования в C#:

  1. Потоки (Threads): Поток - это наименьшая единица выполнения внутри процесса. Он позволяет выполнять код параллельно с другими потоками в том же процессе. В C# потоки могут быть созданы с использованием класса Thread из пространства имен System.Threading.
  2. Синхронизация потоков: Поскольку потоки могут выполняться параллельно, возникает необходимость в синхронизации доступа к разделяемым данным для предотвращения гонок данных и других проблем согласованности. В C# для этого существует несколько механизмов синхронизации, таких как lock, Monitor, Mutex, Semaphore, AutoResetEvent, ManualResetEvent и другие.
  3. Асинхронное программирование: В C# также предоставляются асинхронные операции и ключевые слова async и await, которые позволяют создавать асинхронные методы без использования явных потоков.

Класс Thread. Принципы работы с потоками

Класс Thread в C# из пространства имен System.Threading предоставляет основные средства для работы с потоками. Основные принципы работы с потоками:

  • Синхронизация доступа к разделяемым данным: При работе с разделяемыми данными между потоками необходимо обеспечить правильную синхронизацию для предотвращения гонок данных и других проблем согласованности.
  • Обработка исключений: Необработанные исключения в дополнительных потоках могут привести к непредсказуемым результатам. Необходимо обеспечить корректную обработку исключений в каждом потоке.
  • Управление жизненным циклом потока: Потоки могут быть запущены, остановлены и приостановлены по требованию. Важно правильно управлять их жизненным циклом в зависимости от требований приложения.
Класс Thread предоставляет базовые инструменты для работы с потоками в C#, однако для более удобного и безопасного управления многопоточностью также рекомендуется использовать более высокоуровневые абстракции из пространства имен System.Threading.Tasks, такие как Task и async/await.

Основы параллельного программирования

Параллельное программирование в C# позволяет эффективно использовать ресурсы многоядерных процессоров для ускорения выполнения задач. Вот основы параллельного программирования в C#:

  1. Параллельные циклы и операции: Параллельные циклы: .NET Framework предоставляет класс Parallel для выполнения операций в цикле параллельно. Например, метод Parallel.For позволяет выполнять итерации цикла параллельно.
  2. Задачи и планировщик задач (Task Parallel Library - TPL): Создание и выполнение задач: TPL предоставляет класс Task для создания и выполнения задач параллельно. Метод Task.Run позволяет запускать код в отдельном потоке пула потоков.
  3. Асинхронное программирование с использованием async/await: Создание асинхронных методов: Ключевые слова async и await позволяют создавать асинхронные методы, которые выполняются параллельно без блокировки основного потока.
Параллельное программирование в C# предоставляет множество инструментов для управления параллелизмом и создания эффективных приложений, способных использовать полную мощь многоядерных процессоров.
