Раздел 1

Классы и объекты

Классы и объекты являются основными строительными блоками объектно-ориентированного программирования (ООП). В C# классы определяют типы объектов, а объекты являются экземплярами классов.

Класс — это шаблон для создания объектов, который определяет их поведение (методы) и состояние (данные, переменные).

public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public void Introduce()
    {
        Console.WriteLine($"Hi, my name is {Name} and I am {Age} years old.");
    }
}

Объект — это экземпляр класса, который имеет конкретное состояние и поведение.

Person person = new Person();
person.Name = "Alice";
person.Age = 30;
person.Introduce();

Инкапсуляция объединяет данные и методы, которые работают с этими данными, в единый компонент, называемый классом. Модификаторы доступа (например, `private`, `public`, `protected`, `internal`) контролируют доступ к членам класса.

public class BankAccount
{
    private decimal balance;

    public void Deposit(decimal amount)
    {
        balance += amount;
    }

    public decimal GetBalance()
    {
        return balance;
    }
}

Иерархия классов

Иерархия классов в C# позволяет создавать новые классы на основе существующих, что способствует повторному использованию кода и упрощает его структуру. Это достигается с помощью наследования.

Наследование позволяет создавать новые классы (наследники) на основе существующих (родителей), причем класс-наследник получает все свойства и методы родительского класса, а также может добавлять свои собственные или переопределять существующие.

public class Animal
{
    public string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

Полиморфизм позволяет объектам с одинаковым интерфейсом иметь различную реализацию. Это позволяет использовать один и тот же интерфейс для работы с разными типами объектов, что делает код более гибким и универсальным.

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some sound...");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Meow");
    }
}

Таким образом, полиморфизм позволяет обращаться к объектам через интерфейс базового класса, но при этом вызывать методы производных классов.

Animal myDog = new Dog();
Animal myCat = new Cat();

myDog.MakeSound(); // Output: Bark
myCat.MakeSound(); // Output: Meow
