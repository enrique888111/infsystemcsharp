Принцип наследования

Наследование - один из ключевых принципов объектно-ориентированного программирования (ООП), который позволяет создавать новые классы на основе уже существующих. Наследование позволяет повторно использовать код, упрощать его и создавать более сложные структуры.

В C# наследование достигается с помощью ключевого слова : . Новый класс (наследник) получает все свойства и методы базового класса (родителя).

Пример базового класса и класса-наследника:

public class Animal
{
    public string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

В этом примере класс Dog наследует свойства и методы класса Animal и добавляет собственный метод Bark.

Полиморфизм в ООП

Полиморфизм - это принцип ООП, который позволяет использовать объекты разных классов с единым интерфейсом. Это означает, что один интерфейс может быть использован для работы с объектами разных типов.

В C# полиморфизм достигается через переопределение методов в наследуемых классах с использованием ключевого слова override и виртуальных методов, определенных в базовом классе с использованием ключевого слова virtual.

Пример полиморфизма:

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some sound...");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Meow");
    }
}

Использование полиморфизма:

Animal myDog = new Dog();
Animal myCat = new Cat();

myDog.MakeSound(); // Output: Bark
myCat.MakeSound(); // Output: Meow

В этом примере метод MakeSound переопределен в классах Dog и Cat. При вызове метода MakeSound у объектов типа Animal, но созданных как Dog и Cat, вызываются соответствующие переопределенные методы.
