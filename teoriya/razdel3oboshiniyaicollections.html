Раздел 3

Обобщения. Обобщенные классы. Иерархии обобщенных классов

Обобщения (Generics) в C# позволяют создавать классы, структуры, методы и интерфейсы, работающие с различными типами данных без необходимости повторного создания кода.

Обобщенные классы могут быть параметризованы типами данных, что делает их более гибкими и универсальными. Использование обобщений позволяет писать более безопасный и эффективный код.

Пример обобщенного класса:

public class GenericList<T>
{
    private List<T> list = new List<T>();

    public void Add(T value)
    {
        list.Add(value);
    }

    public T Get(int index)
    {
        return list[index];
    }
}

Иерархии обобщенных классов позволяют создавать иерархию классов, используя обобщения. Например:

public class MyBaseClass<T>
{
    // implementation
}

public class MyDerivedClass<T> : MyBaseClass<T>
{
    // implementation
}

Коллекции. Типы коллекций. Необобщенные коллекции. Обобщенные коллекции

Коллекции в C# представляют собой структуры данных, которые позволяют хранить и управлять группами объектов. Они предоставляют различные способы организации данных для более эффективной работы с ними.

В C# существуют различные типы коллекций, включая необобщенные коллекции (например, ArrayList, Hashtable) и обобщенные коллекции (например, List, Dictionary).

Пример использования обобщенной коллекции List:

List<int> numbers = new List<int>();
numbers.Add(1);
numbers.Add(2);
numbers.Add(3);

foreach (int number in numbers)
{
    Console.WriteLine(number);
}

Обобщенные коллекции предпочтительны, так как они обеспечивают типобезопасность и более высокую производительность по сравнению с необобщенными коллекциями.
