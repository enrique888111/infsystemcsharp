Раздел 2

Система типов

В C# система типов играет ключевую роль в обеспечении безопасности типов и управлении памятью.

Значимые типы включают простые типы данных, такие как целые числа (int, long), числа с плавающей точкой (float, double), символы (char), логические значения (bool) и структуры (struct). Они хранятся в стеке, что обеспечивает быстрый доступ к данным и эффективное управление памятью.

Ссылочные типы включают классы (class), интерфейсы (interface), массивы и делегаты. Они хранятся в куче и содержат ссылки на данные. Это позволяет им быть гибкими и динамически изменяемыми, но требует дополнительного управления памятью CLR.

Структура программы. Переменные, операции и выражения

Структура программы в C# включает пространства имен (namespace), классы (class), методы и другие элементы. Пространства имен используются для организации кода и предотвращения конфликтов имен.

Переменные в C# являются именованными областями памяти, предназначенными для хранения данных. Они могут быть объявлены с указанием типа данных и могут содержать различные значения, включая примитивные типы, объекты и ссылки.

Операции включают арифметические, логические, побитовые операции и многое другое. Они позволяют выполнять различные действия с данными, такие как сложение, умножение, сравнение и т. д.

Выражения в C# представляют собой комбинации переменных, литералов и операторов, которые вычисляются в конкретное значение. Они являются основными строительными блоками программы и используются для выполнения различных операций и принятия решений.

Сборки. Атрибуты. Директивы. Пространства имен

Сборки в C# представляют собой исполняемые файлы (.exe) или библиотеки классов (.dll), содержащие метаданные и IL-код (промежуточный язык), который CLR преобразует в машинный код во время выполнения.

Атрибуты представляют собой специальные конструкции, добавляемые к элементам программы для предоставления дополнительной информации компилятору или среде выполнения. Например, атрибут [Serializable] указывает, что объект может быть сериализован.

Директивы предварительной обработки (препроцессорные директивы) позволяют включать или исключать определенные части кода в зависимости от условий компиляции. Это может быть полезно для условной компиляции различных версий программы или для включения отладочной информации.

Пространства имен используются для организации кода и предотвращения конфликтов имен между различными компонентами программы. Они облегчают разработку и поддержку программы, разделяя ее на логические модули.

Операторы

Операторы в C# представляют собой специальные символы или ключевые слова, которые выполняют определенные операции над данными.

Арифметические операторы (+, -, *, /, %) используются для выполнения арифметических операций, таких как сложение, вычитание, умножение и деление.

Операторы сравнения (!=, >, <, >=, <=) используются для сравнения двух значений и возвращают логическое значение true или false в зависимости от результата сравнения.

Логические операторы (&&, ||, !) используются для выполнения логических операций над булевыми значениями.

Побитовые операторы (&, |, ^, ~, <<, >>) используются для выполнения побитовых операций над целыми числами.

Операторы присваивания (=, +=, -= и т. д.) используются для присваивания значения переменной.

Условный оператор (?:) используется для выполнения условных операций на основе значения булевого выражения.

Процедуры и функции – методы класса. Встроенные методы

Методы в C# представляют собой блоки кода, выполняющие определенную операцию или задачу. Они могут принимать аргументы и возвращать значения.

Встроенные методы предоставляются в стандартной библиотеке классов .NET Framework и могут использоваться для выполнения различных операций, таких как вывод на консоль, математические вычисления, обработка строк и т. д.

Пример


        try
        {
            // Код, который может вызвать исключение
        }
        catch (Exception ex)
        {
            // Обработка исключения
        }
        finally
        {
            // Код, который выполняется в любом случае
        }

В блоке try размещается код, который может вызвать исключение. В блоке catch определяется обработка исключения. Блок finally содержит код, который будет выполнен в любом случае, независимо от того, возникло исключение или нет.

Классы

В C# классы представляют собой основные строительные блоки объектно-ориентированного программирования. Класс определяет состояние (поля) и поведение (методы) объектов.

Классы в C# могут содержать следующие элементы:

  • Поля (Fields): переменные, объявленные внутри класса, которые представляют его состояние.
  • Методы (Methods): функции, определенные внутри класса, которые представляют его поведение. Методы могут быть публичными, приватными, защищенными и т. д., в зависимости от уровня доступа.
  • Свойства (Properties): специальные методы доступа к полям класса, которые обеспечивают инкапсуляцию и контроль доступа к данным.
  • Конструкторы (Constructors): специальные методы, вызываемые при создании объекта класса. Используются для инициализации объекта и его состояния.
  • Деструкторы (Destructors): методы, вызываемые перед уничтожением объекта. Используются для освобождения ресурсов, занятых объектом.
  • Индексаторы (Indexers): специальные методы, позволяющие обращаться к объекту класса, как к массиву.
  • События (Events): механизм, позволяющий классу уведомлять другие классы о возникновении определенных событий.
  • Вложенные классы (Nested Classes): классы, определенные внутри других классов. Используются для организации и структурирования кода.

Массивы. Класс Array

Массивы в C# представляют собой упорядоченные коллекции элементов одного типа, доступ к которым осуществляется по индексу.

Основные характеристики массивов в C#:

  • Типизация: все элементы массива должны быть одного типа данных.
  • Фиксированный размер: массивы имеют фиксированную длину, которая определяется при создании.
  • Индексация: доступ к элементам массива осуществляется по их индексу, начиная с нуля.
  • Многомерные массивы: C# поддерживает многомерные массивы, включая одномерные, двумерные и многомерные массивы.

Класс Array предоставляет различные методы для работы с массивами, такие как сортировка (Sort()), поиск (IndexOf()), копирование (Copy()) и т. д.

Символы и строки постоянной длины. Классы String и StringBuilder

В C# символы (char) представляют собой одиночные символы Unicode, которые хранятся в двух байтах памяти. Они используются для представления букв, цифр и других символов.

Строки (string) представляют собой последовательности символов Unicode переменной длины. В отличие от символов, строки являются неизменяемыми (immutable), что означает, что их содержимое нельзя изменить после создания. Для выполнения операций изменения содержимого строки используется класс StringBuilder, который представляет собой изменяемую последовательность символов.

Класс String предоставляет множество методов для работы со строками, таких как конкатенация (Concat()), поиск (IndexOf()), разделение (Split()) и т. д.

Регулярные выражения

Регулярные выражения (регулярные выражения) представляют собой мощный инструмент для поиска и обработки текста на основе шаблонов.

В C# регулярные выражения реализованы с использованием класса Regex из пространства имен System.Text.RegularExpressions. С их помощью можно выполнять такие операции, как поиск шаблонов в тексте, замена текста на основе шаблона, разделение текста на подстроки и многое другое.

Пример использования регулярных выражений в C#:


        string input = "Hello 123 World";
        string pattern = @"\d+";
        bool match = Regex.IsMatch(input, pattern); // true
Структуры и Перечисления

Структуры. Перечисления

Структуры (struct) в C# представляют собой типы значений, аналогичные классам, но с некоторыми отличиями. Они обычно используются для представления небольших и легких объектов, которые имеют семантику значений.

Перечисления (enum) представляют собой набор именованных констант, которые обеспечивают более читаемый и понятный код. Они используются для определения типов данных, которые могут принимать только определенные значения из набора.

Пример использования структур и перечислений:


        public struct Point
        {
            public int X;
            public int Y;
        }
        
        public enum Days
        {
            Sunday,
            Monday,
            Tuesday,
            Wednesday,
            Thursday,
            Friday,
            Saturday
        }

Отношение между классами

В C# отношения между классами могут быть выражены различными способами, включая наследование, композицию и агрегацию. Наследование (Inheritance): это механизм, позволяющий классу (производному классу) наследовать члены другого класса (базового класса). Производный класс может использовать и расширять функциональность базового класса. Например:


                public class Animal
                {
                    public void Eat() { }
                    public void Sleep() { }
                }
                
                public class Dog : Animal
                {
                    public void Bark() { }
                }

Интерфейсы

Интерфейсы в C# представляют собой контракт, который определяет набор методов и свойств, которые класс должен реализовать. Они позволяют определить общий интерфейс для различных классов, что обеспечивает гибкость и расширяемость кода.


                public interface ILogger
                {
                    void Log(string message);
                }
                
                public class ConsoleLogger : ILogger
                {
                    public void Log(string message)
                    {
                        Console.WriteLine(message);
                    }
                }

Обработка исключительных ситуаций

Исключения в C# представляют собой ошибочные ситуации, которые могут возникнуть во время выполнения программы. Обработка исключений позволяет управлять такими ситуациями и принимать соответствующие действия для восстановления или завершения программы.

Пример:


                try
                {
                    // Код, который может вызвать исключение
                }
                catch (Exception ex)
                {
                    // Обработка исключения
                }
                finally
                {
                    // Код, который выполняется в любом случае
                }

В блоке try размещается код, который может вызвать исключение. В блоке catch определяется обработка исключения. Блок finally содержит код, который будет выполнен в любом случае, независимо от того, возникло исключение или нет.
